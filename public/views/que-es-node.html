Introducción a node

¿Que no es node?

  • No es un lenguaje
  • No es una libreria
  • No es un framework
  • No es una sintaxis de JavaScript
  • No es un servidor

¿Qué es node?

  • Los navegadores tienen un motor que interpreta JavaScript (cuando nosotros escrbimos JS y lo ejecutamos en el navegador dentro del navegador hay un intérprete que se encarga de leer eso y de traducirlo a código que pueda entender el navegador).
  • El motor más famoso es el V8 desarrollado por Google en C++
  • En 2009 Ryan Dahl, extrae ese motor y hace que se ejecute de forma externa al navegador
  • Node es un interprete de JavaScript que nos permite ejecutar JavaScript sin necesidad de un navegador (es tomar el intérprete y poder ponerlo en nuestra máquina o en cualquier otro sitio y poder escribir código JS sin tener que tener un navegador abierto que lo interprete).

Cuando esto se presentó en 2009 empezó lo que se denominaría una revolución en el mundo del desarrollo web porque empezarón a surgir un monton de librerias y un monton de módulos que nos ayudan en el desarrollo web y todo esto es gracias a poder ejecutar JS fuera del navegador.

¿Quién utiliza Node?

  • Netflix
  • Uber
  • Nasa
  • Paypal
  • trello
  • Otros cientos más..

A día de hoy no se concibe el desarrollo web sin Node, no solo en el lado del servidor que es (opcional), si no Node en la parte de desarrollo.

Cuando nosotros estamos desarrollando aplicaciones web, Node nos ayuda muchisimo y a día de hoy es prácticamente imposible ir a trabajar a un sitio y que no se utilize Node.

¿Como funciona Node?

  • Cuando nosotros instalamos node, instalamos el núcleo solamente ( Es decir el intérprete de Js, que nos permite hacer muchas cosas pero hay ciertas cosas que necesitamos ayuda externa de los módulos de node que se verán más adelante.)
  • Para añadir funcionalidades extra, node nos da un gestor de paquetes . Si hemos escuchado npm ese es el gestor de paquetes que trae node integrado, que para que nos demos una idea sería como el play store de google, nosotros tenemos el celular pero si queremos añadirle funcionalidades extra necesitariamos un sitio de donde descargar esas funcionalidades, y en el caso de node esas funcionalidades las tenemos a través de npm.

¿Qué es NPM y Yarn?

Son dos alternativas que tenemos

  • La principal diferencia es que npm ya viene integrado con node una vez que lo instalamos ya tenemos acceso a ese gestor de paquetes.
  • yarn es más rápido que npm pero como yarn no viene integrado lo tenemos que instalar aparte.
  • Sea cual sea el que usemos debemos recordar usar siempre el mismo para evitar conflictos. Así que si empezamos un proyecto con npm debemos utilizar npm a lo largo de todo el proyecto
